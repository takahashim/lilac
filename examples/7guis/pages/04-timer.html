4. Timer

Elapsed time tracks a setInterval (auto-cancelled when the component unmounts). The duration slider can extend a finished timer to make it resume.

Source

How it works

every(100) { … } で setInterval を回し、component unmount 時に Lilac が自動 cancel する。残り時間 / progress は computed で導出するので、duration を変えれば完了済みでも自然に再開する。

  • every(ms) { … } — repeating timer。返値の Timer#stop で早期中断可
  • 明示の clearInterval 不要 — cleanup は Lilac が握る
  • data-attr-value="@progress" — <progress> の value 属性を Signal で driving
  • data-bind="@duration_sec" — range slider と Signal の双方向 binding
  • elapsed が duration に達したら止めるだけで break しない → 後から duration を伸ばすと自然に resume

every(100) { … } drives the setInterval; Lilac auto-cancels it on unmount. Remaining time / progress are derived via computed, so raising the duration naturally resumes a finished timer.

  • every(ms) { … } — repeating timer; returns a Timer#stop-able handle
  • No explicit clearInterval — Lilac owns the cleanup
  • data-attr-value="@progress" — drives the <progress> value attribute
  • data-bind="@duration_sec" — two-way binds the range slider
  • The loop never breaks; elapsed simply stops at duration, so extending duration later resumes naturally
