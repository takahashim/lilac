How it works

あえて data-bind による双方向 binding は使わず、refs.X.on(:input) で他方の input に直接書き込む。Signal を介さないので feedback loop が原理的に起きない。

  • refs.celsius.on(:input) { … } — DOM event listener。unmount で自動 remove
  • refs.celsius.value — DOM input の value を直接 read/write(RefElement#PROPS)
  • Float(s) rescue nil — 数値パース失敗を防御
  • 双方向 binding が必要に見える場面でも、event-driven のほうがシンプルなことは多い

Deliberately skip two-way data-bind: refs.X.on(:input) writes straight to the partner input. With no Signal round-trip, a feedback loop is impossible by construction.

  • refs.celsius.on(:input) { … } — DOM event listener; auto-removed on unmount
  • refs.celsius.value — read/write the input's value directly (RefElement#PROPS)
  • Float(s) rescue nil — guards against parse failure
  • Even when two-way binding looks needed, event-driven is often simpler
