How it works

computed で「各 field が valid か」「Book できるか」を Signal 群から導出し、data-attr-disabled / data-class で submit ボタン活性化や error 表示に直結させる。

  • data-bind="@kind" — <select> の選択値を Signal に双方向 bind
  • computed { … } — 依存 Signal を自動追跡する read-only Signal
  • data-attr-disabled="@cannot_submit" — Signal が truthy なら disabled attribute が立つ
  • data-class="{ 'is-invalid': @start_invalid }" — Signal が truthy なら class 追加
  • ISO 日付文字列(YYYY-MM-DD)は辞書順 = 時系列順なので Date パース不要

Use computed to derive per-field validity and overall bookability from the underlying Signals; data-attr-disabled / data-class then drive the submit button and error styling directly.

  • data-bind="@kind" — two-way binds the <select> value
  • computed { … } — read-only Signal that auto-tracks dependencies
  • data-attr-disabled="@cannot_submit" — sets disabled when the Signal is truthy
  • data-class="{ 'is-invalid': @start_invalid }" — adds the class conditionally
  • ISO date strings (YYYY-MM-DD) sort lexicographically the same way as chronologically — no Date parsing needed
