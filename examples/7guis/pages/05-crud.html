5. CRUD

Prefix-filter a list, select a row to load the inputs, then create / update / delete. The row is a separate component that looks up shared state via expose / lookup.

Source

How it works

list は data-each + key で render し、行 component は prop 宣言だけで iteration item の同名 field を auto-fill 受け取る。親子間の共有 state は expose / lookup。

  • data-each="@visible" data-key="id" — Signal 配列を key 付きで render(行再利用が効く)
  • prop :name, String — Signal として @name ivar と name accessor を生成。data-each 内では item の同名 field から auto-fill(prop auto-fill, §7.6)
  • expose :selected_id, @selected_id(prepare_setup 内) — 子の lookup(:selected_id) から参照される
  • 行の click は root.dispatch(:row_chosen, bubbles: true) で親へ。親側で component_for(event[:target]) → row の id accessor で解決
  • filter 入力も data-bind="@filter" + computed の組み合わせで list 表示が即追従

The list renders with data-each + key; each row component receives its iteration item's same-named fields automatically through prop declarations. Shared parent / child state uses expose / lookup.

  • data-each="@visible" data-key="id" — renders the Signal array with stable row reuse
  • prop :name, String — exposes @name as a Signal and name accessor; inside data-each it auto-fills from the current item's name field (prop auto-fill, §7.6)
  • expose :selected_id, @selected_id (inside prepare_setup) — children read it via lookup(:selected_id)
  • Row clicks root.dispatch(:row_chosen, bubbles: true); the parent resolves the row via component_for(event[:target]) and reads its id accessor
  • The filter input is data-bind="@filter" + computed; the list updates as you type
