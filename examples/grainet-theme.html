provide / inject — theme demo

The outer theme-app widget publishes a :theme signal in its provides hook. Each theme-card and theme-badge widget calls inject(:theme) in its setup and binds its CSS classes reactively. None of the inner widgets know who provides the theme — they just ask for it by key.

First card

This card asks inject(:theme). Two levels deep, the badge does the same independently. status

Second card

Both cards re-render only their CSS class on toggle — the DOM nodes are not replaced, so any focus / scroll / nested widget state is preserved.
