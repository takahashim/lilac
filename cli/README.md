# grainet-cli

Build tool for **Grainet single-file components** (`.gnt`) — template +
Ruby script in one file, compiled to static HTML for the Grainet component
runtime on mruby-wasm.

> [!WARNING]
> Pre-alpha. API and CLI surface will change.

## What is a `.gnt` file?

A single-file component containing markup, optional sub-templates, and a
Ruby script that defines a `Grainet::Component` subclass:

```html
<!-- components/counter.gnt -->
<template>
  <div data-component="counter">
    <button data-ref="inc">+</button>
    <span data-ref="count">0</span>
  </div>
</template>

<!-- Sub-templates use `data-template="..."` — same convention the
     Grainet runtime uses for `bind_list` clone-targets. -->
<template data-template="counter-banner">
  <div class="banner"></div>
</template>

<script type="text/ruby">
  class Counter < Grainet::Component
    def setup
      @count = signal(0)
      refs.inc.on(:click) { @count.update(&:succ) }
      bind refs.count, text: @count
    end
  end
</script>
```

`.gnt` is HTML5-valid (use a `.gnt → text/html` association in your
editor for syntax highlighting). The build tool extracts the top-level
`<template>` and `<script type="text/ruby">` blocks; everything inside
each block is preserved verbatim.

`grainet build` reads `components/*.gnt` + `pages/*.html`, inlines the
referenced component templates and scripts into each page, and writes the
result to `dist/`.

The kebab-to-CamelCase autoregister feature in Grainet means
`<div data-component="counter"></div>` resolves to the `Counter` class with
no `Grainet.register` call needed.

## Installation

```sh
gem install grainet-cli
```

Or in a Gemfile:

```ruby
group :development do
  gem "grainet-cli"
end
```

Requires Ruby 3.2 or later.

## Usage

```sh
grainet new my-app         # scaffold a fresh project (Counter sample inside)
cd my-app && bundle install

grainet doctor             # check project setup (runtime, component refs, paths)

grainet build              # produce dist/ from components/ + pages/
grainet build --output _site

grainet dev                # build + serve + live reload at http://127.0.0.1:5173
grainet dev --port 8000    # bind to a different port
```

### Configuration

Project-wide defaults can live in `grainet.config.rb` at the project
root (a commented template is generated by `grainet new`):

```ruby
Grainet::CLI.configure do |c|
  c.components_dir = "src/components"
  c.pages_dir   = "src/pages"
  c.dev_port    = 3000
end
```

Precedence (lowest to highest): built-in defaults → `grainet.config.rb`
→ CLI flags (`--components`, `--port`, etc.).

`grainet dev` watches `components/**/*.gnt` and `pages/**/*.html` for
changes, rebuilds on save, and pushes a `reload` event over SSE to every
connected browser tab.
